# Contributing to Blokdoc

Thank you for your interest in contributing to Blokdoc! We welcome contributions from everyone.

## How to Contribute

1. **Fork the Repository**
   - Fork the repository to your GitHub account.

2. **Clone the Repository**
   ```
   git clone https://github.com/your-username/Blokdoc.git
   cd Blokdoc
   ```

3. **Create a Branch**
   ```
   git checkout -b feature/your-feature-name
   ```

4. **Make Changes**
   - Make your changes to the codebase.
   - Follow the coding style and guidelines.

5. **Run Tests**
   ```
   npm test
   ```

6. **Commit Changes**
   ```
   git add .
   git commit -m "Add feature: description of your changes"
   ```

7. **Push to Your Fork**
   ```
   git push origin feature/your-feature-name
   ```

8. **Create a Pull Request**
   - Go to the original Blokdoc repository on GitHub.
   - Click on "New Pull Request".
   - Select your fork and branch.
   - Add a description of your changes.
   - Submit the pull request.

## Code Style

- Use TypeScript for type safety.
- Follow ESLint rules.
- Use meaningful variable and function names.
- Write comments for complex logic.
- Create comprehensive tests for new features.

## Pull Request Guidelines

- Keep PRs focused on a single change.
- Update documentation if necessary.
- Add tests for new features.
- Make sure all tests pass.
- Reference any relevant issues in your PR description.
